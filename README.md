## 📖 الديباجة والرؤية (Vision & Mission)
وُلدت فكرة **"مقرري+"** لحل معضلة وطنية وعالمية: **الفجوة بين المناهج الأكاديمية ومتطلبات سوق العمل المتسارعة**. النظام ليس مجرد منصة تعليمية لعرض المقررات، بل هو **"نظام بيئي متكامل" (Ecosystem)** وشبكة معارف (Knowledge Graph) تسمح لخبراء الصناعة المعتمدين بالدخول وحقن المقررات الجامعية بمهارات وأدوات سوق العمل الحقيقية، لتوليد كفاءات هندسية جاهزة للتوظيف الفوري. تم تصميم هذه المعمارية لتكون نموذجاً قياسياً ينطلق من كليات الهندسة وعلوم الحاسب ليخدم المنظومة التعليمية بأكملها.

---

## 🚀 المميزات القاتلة (The Killer Features)

1. **السيرة الذاتية المدفوعة بالمهارات (Skills-Driven CV - B2C):**
   * لا يكتب الطالب أنه "درس مقرر هندسة البرمجيات" بل يقوم النظام آلياً باستخراج (المهارات، الأدوات، والشهادات الاحترافية) من إثراءات الخبراء للمقررات التي سجل بها الطالب، ويولد له سيرة ذاتية تقنية جاهزة للطباعة.
2. **مؤشر الفجوة الأكاديمية (Academic Gap Index - B2B):**
   * لوحة قيادة تحليلية للإدارة وعمداء الكليات مبنية بـ `Chart.js`، تعرض إحصائيات حية حول أكثر المهارات والأدوات التي يطلبها الخبراء حالياً، مما يسهل تحديث المناهج ببيانات حقيقية.
3. **مركز قيادة الخبير والمحرر الحي (Expert Command Center & Live Editor):**
   * لوحة تحكم للخبير تقيس "أثره الطلابي"، مع محرر نصوص (Markdown) منقسم الشاشة يظهر التنسيق بالوقت الفعلي.
4. **نظام التلعيب والتفاعل (The Gamification Loop):**
   * دورة تفاعل حية تتيح للطلاب رفع تقييم الإثراءات الممتازة (Upvote) وحفظها في مفضلتهم (Bookmark) للعودة إليها.
5. **محرك البحث الذكي الشامل (Global Smart Search):**
   * محرك بحث مركزي مدعوم باستعلامات `JPQL` معقدة، يبحث بالتوازي داخل (أسماء المقررات، نصوص الإثراءات، أسماء الخبراء، والمهارات التقنية).

## 🏛️ الهيكلة المعمارية (The Knowledge Graph Architecture)
تم بناء قاعدة البيانات لتتجاوز الجداول المسطحة وتصبح شبكة علاقات معقدة (Many-to-Many):
* **المستعمرة الأكاديمية:** `University` -> `College` -> `Department` -> `Course`.
* **المستعمرة الصناعية:** `Industry` -> `Skill` & `Tool` & `ProfessionalCertificate`.
* **نقطة التصادم (The Nexus):** كيان `CourseEnrichment` الذي يربط (المقرر + الخبير + المهارات + الأدوات + الشهادات + المحتوى النصي + تفاعلات الطلاب).

---

## 🗄️ محرك الحقن المعرفي (Massive Data Seeder)
تم تزويد المنصة بمحرك توليد بيانات (Seeder) يحقن قاعدة البيانات لحظة الإقلاع ببيانات حقيقية من السوق السعودي والعالمي لحل مشكلة "البداية الباردة"، ويشمل:
* **73 مقرراً أكاديمياً** متكاملاً.
* **15 شهادة مهنية عالمية** (مثل: PMP, AWS Certified, CISSP).
* **40 مهارة وأداة برمجية** حديثة.
* **8 خبراء محاكين** بمناصب قيادية في جهات سيادية وتقنية (مثل: سدايا، أرامكو، stc، الهيئة الوطنية للأمن السيبراني، علم) مع نصوص إثرائية (Markdown) مطولة تربط النظرية بالتطبيق.

## 🛠️ الترسانة التقنية (Full Tech Stack)
| الفئة | التقنية المستخدمة | الدور التشغيلي |
| :--- | :--- | :--- |
| **لغة البرمجة الأساسية** | **Java 17 (LTS)** | توفير بيئة تشغيل مستقرة وهندسة الكيانات (OOP). |

| **المكتبات المساعدة** | **Chart.js & Marked.js** | رسم المخططات البيانية، وتحويل الـ Markdown حياً. |

## ⚙️ بروتوكول التجهيز والإقلاع الميداني (Deployment Protocol)

### 1. الإقلاع المحلي (Local Development):
1. قم بتهيئة `MySQL` (عبر XAMPP مثلاً) وأنشئ قاعدة بيانات فارغة باسم `muqarari_plus` بترميز `utf8mb4_general_ci`.
2. افتح موجه الأوامر داخل المشروع ونفذ:
   ```bash
   mvn clean spring-boot:run
   (سيقوم النظام آلياً ببناء الجداول وحقن البيانات الحقيقية بالكامل).

2. الإقلاع السحابي (Cloud Production):
   المنصة مجهزة بوضع (Chameleon Mode) في ملف application.properties.

جاهزة للرفع المباشر على خوادم Railway.app أو Heroku؛ حيث تلتقط المتغيرات السحابية (MYSQL_URL, PORT) آلياً وتعمل دون أي تعديل في الكود.

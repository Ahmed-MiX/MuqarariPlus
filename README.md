# 🎓 منصة مقرري+ (Muqarari+) - التوثيق التقني والتشغيلي الشامل
> **"الهندسة المعمارية لربط النظرية الأكاديمية بالتطبيق الميداني - إصدار الهاكاثون الذهبي v1.5"**

## 📖 الديباجة والرؤية (Vision & Mission)
تمت هندسة وبناء منصة **"مقرري+"** كحل تقني جذري لسد الفجوة بين المقررات الجامعية النظرية لطلاب كليات الهندسة وعلوم الحاسب، وبين المتطلبات الحقيقية لسوق العمل. النظام ليس مجرد واجهات عرض، بل هو **نظام إدارة محتوى ديناميكي (Dynamic CMS)** معقد، مبني على معمارية (4-Tier Role-Based Security) لضمان تدفق المعرفة من خبراء الصناعة إلى الطلاب ببيئة آمنة وموثوقة.

---

## 🛠️ الترسانة التقنية (Full Tech Stack)
| الفئة | التقنية المستخدمة | الدور التشغيلي |
| :--- | :--- | :--- |
| **لغة البرمجة الأساسية** | **Java 17 (LTS)** | توفير بيئة تشغيل مستقرة ودعم السجلات (Records). |
| **إطار العمل الخلفي (Backend)** | **Spring Boot 3.2.4** | إدارة الحاويات (IoC) وحقن التبعيات (Dependency Injection). |

| **قواعد البيانات** | **MySQL 8.x (MariaDB via XAMPP)** | الخزنة المركزية لتخزين نصوص الموقع، المقررات، والمستخدمين. |

| **إدارة الاعتمادات** | **Maven 3.x** | جلب المكتبات، التنظيف (`clean`)، وبناء المشروع (`build`). |

## ⚙️ التجهيز العسكري لبيئة التطوير (Prerequisites Setup)
يُحظر البدء في تشغيل المشروع قبل التأكد من استيفاء كافة المتطلبات التالية بدقة:
1. **Java Development Kit (JDK) 17:** تثبيت النسخة وضبط مسار المتغير البيئي `JAVA_HOME` في نظام التشغيل.
2. **بيئة التطوير (IDE):** استخدام **IntelliJ IDEA** (أو Antigravity) لتوفير مزامنة حية وتحليل ذكي للأخطاء.
3. **الخادم المحلي (XAMPP):** تثبيت حزمة XAMPP (إصدار يدعم PHP 8.x) لتوفير خادمي `Apache` و `MySQL`.
4. **Git:** لإدارة مستودع الأكواد والتحكم في الإصدارات (Version Control).

## 🚀 بروتوكول التثبيت والإقلاع (Deployment Protocol)

### المرحلة الأولى: تهيئة الخزنة المركزية (Database Setup)
1. افتح لوحة تحكم **XAMPP Control Panel**.
2. قم بتشغيل خدمة **Apache** (يجب أن يضيء باللون الأخضر).
3. قم بتشغيل خدمة **MySQL** (يجب أن يضيء باللون الأخضر).
   *(ملاحظة: إذا كان MySQL يعمل مسبقاً في خلفية الويندوز، شغل Apache فقط).*
4. افتح المتصفح على الرابط: `http://localhost/phpmyadmin`.
5. قم بإنشاء قاعدة بيانات فارغة تماماً باسم: **`muqarari_plus`**.
6. **أمر حرج:** يجب تعيين الترميز (Collation) إلى `utf8mb4_general_ci` لضمان عدم تشوه الحروف العربية.

### المرحلة الثانية: استنساخ وتطهير المشروع (Clone & Clean)
1. افتح سطر الأوامر (Terminal) ونفذ الاستنساخ: `git clone [رابط المستودع]`.
2. افتح المجلد في **IntelliJ IDEA** وانتظر حتى ينتهي Maven من تحميل كافة التبعيات (Dependencies).
3. افتح الـ Terminal داخل المحرر ونفذ أمر التطهير الشامل:
   ```bash
   mvn clean
   ```

### المرحلة الثالثة: الإقلاع الذهبي (The Golden Boot)
1. في نفس الـ Terminal، نفذ أمر التشغيل الدقيق:
   ```bash
   mvn spring-boot:run
   ```
2. **المراقبة أثناء الإقلاع:** سيقوم النظام بـ:
   * بناء الجداول (DDL Auto Update).
   * استدعاء `DatabaseSeeder` لزرع **4 حسابات**، و **73 مقرراً حقيقياً** (لكلية CCIS).
   * زرع **163 صفاً** في جدول `site_content` لتشغيل النظام الديناميكي.
3. الوصول للموقع: افتح `http://localhost:8080`.

## 🗄️ دليل الإدارة الديناميكية: التعديل الحي عبر MySQL
المشروع لم يعد يعتمد على ملفات ثابتة، بل هو نظام CMS متكامل. لتعديل أي نص في الموقع بدون لمس الكود البرمجي:
1. تأكد من عمل **Apache** و **MySQL** في XAMPP.
2. افتح الرابط: `http://localhost/phpmyadmin`.
3. ادخل على قاعدة البيانات `muqarari_plus` ثم اضغط على جدول **`site_content`**.
4. **تجاوز قيد الـ 25 سطراً:** واجهة phpMyAdmin تعرض 25 سطراً فقط افتراضياً. لرؤية كافة البيانات (163 سطراً)، اذهب إلى مربع "عدد الأسطر" (Number of rows) أعلى الجدول، وافتح القائمة المنسدلة، واختر **250**.
5. ابحث عن المفتاح الذي تريد تغييره (مثلاً `hero.heading`).
6. اضغط مرتين على النص في عمود `value_ar` (أو الإنجليزي).
7. اكتب النص الجديد واضغط `Enter`.
8. ارجع للموقع `localhost:8080`، واضغط تحديث (Refresh).. **سيظهر التعديل فوراً!**

## 📜 سجل التطور المعماري (Architectural Evolutionary Journey)

* **v1.0 (التأسيس):** بناء الكيانات (Entities) والربط بقاعدة البيانات. الواجهات كانت تعتمد على نصوص إنجليزية صلبة (Hardcoded) لا تقبل التعريب.
* **v1.1 (أزمة اللواحق):** محاولة التعريب عبر نظام `MessageSource` بإضافة لواحق `_ar` و `_en` للمفاتيح. أدى ذلك لانهيار محرك Thymeleaf وظهور رموز `??key_ar??`.
* **v1.2 (تصادم الفاصولياء - Bean Conflict):** محاولة بناء نظام تبديل لغات بإنشاء `LocaleConfig`. اصطدم مع `WebConfig` مما أدى لانهيار الإقلاع بخطأ `BeanDefinitionOverrideException`.
* **v1.3 (التطهير الشامل):** مسح 10 ملفات HTML تماماً من اللواحق المشوهة. توحيد المعمارية تحت `WebConfig` واحد، وإصلاح ملفات الـ `.properties`.
* **v1.5 (النسخة الذهبية - Dynamic CMS):** إلغاء الاعتماد على الملفات الثابتة. إنشاء كيان `SiteContent` وتطوير `GlobalAttributeAdvice` ليقوم بحقن كل نصوص الموقع (عبر `ContentService`) إلى الواجهات من MySQL مباشرة في كل طلب HTTP.

## ⚠️ غرفة العمليات (The War Room): السجل الشامل للأخطاء والحلول
هذا السجل يوثق كل عقبة تقنية واجهتنا منذ السطر الأول في المشروع وحتى لحظة الإطلاق، ليكون مرجعاً لأي مهندس يواجه مواقف مشابهة:

| # | المشكلة التقنية (The Error) | التشخيص والسبب الجذري | الحل الهندسي الصارم |
| :-: | :--- | :--- | :--- |
| **1** | `ModelParseException` | نسخ أوامر الـ Terminal ولصقها بالخطأ داخل هيكل ملف `pom.xml`. | التطهير اليدوي لملف الـ XML وإعادة تحميل الـ Maven `Reload Project`. |
| **2** | `Failed to delete target` | فشل أمر `mvn clean` لوجود قفل (Lock) على ملفات `target` من قبل عملية Java تعمل بالخفاء. | استخدام موجه الأوامر كمسؤول وتنفيذ: `taskkill /F /IM java.exe`. |
| **3** | `??login.title_ar??` | محرك Thymeleaf يبحث عن مفتاح به لاحقة `_ar` داخل قاموس الترجمة ولا يجده. | التطهير المعماري: مسح اللواحق من قوالب HTML واستخدام المفتاح المجرد `#{login.title}`. |
| **4** | `BeanDefinitionOverrideException` | تعريف `localeResolver` في ملفين مختلفين (`WebConfig` و `LocaleConfig`) مما أدى لتصادم Beans. | حذف ملف `LocaleConfig` مادياً، ودمج معمارية اللغات داخل `WebConfig`. |
| **5** | **AI Hallucination (كذب النموذج)** | الذكاء الاصطناعي يدعي أنه قام بتعديل الملفات على القرص، بينما هو يكتب الأكواد في الدردشة فقط. | تفعيل "بروتوكول الإعدام البرمجي": إرسال أوامر صارمة تجبر أداة الـ API على التنفيذ المادي (File Write). |
| **6** | **Phantom IDE Buffer** | المحرر يحتفظ بنسخة "شبحية" (Unsaved Buffer) من `LocaleConfig` رغم حذفه من القرص الصلب. | إغلاق التبويب من أعلى المحرر واختيار `Don't Save` لمنع إعادة كتابة الملف القاتل. |
| **7** | `ERR_CONNECTION_REFUSED` | محاولة فتح `phpMyAdmin` في المتصفح قبل تشغيل خوادم XAMPP. | فتح لوحة XAMPP وتشغيل خدمتي `Apache` و `MySQL` للون الأخضر أولاً. |
| **8** | `Port 3306 in use` | محاولة تشغيل MySQL من XAMPP، بينما هناك نسخة MySQL أصلية تعمل كخدمة في خلفية الويندوز. | تجاهل رسالة الخطأ وتشغيل `Apache` فقط، حيث أن المشروع متصل فعلياً بالقاعدة التي في الخلفية. |
| **9** | `Access denied for user (using password: NO)` | `phpMyAdmin` يحاول الدخول للقاعدة الخفية بدون كلمة مرور، بينما القاعدة مشفرة وتتطلب واحدة. | الدخول لملف `config.inc.php` الخاص بـ XAMPP وتعديل `auth_type` من `config` إلى `cookie`. |
| **10** | `Could not find goal 'Run'` | خطأ مطبعي قاتل (Typo) بكتابة حرف كبير: `mvn clean spring-boot:Run`. | الالتزام بحساسية حالة الأحرف (Case Sensitivity) وكتابة الأمر: `spring-boot:run`. |
| **11** | `Access denied for user 'pma'` | رسالة حمراء سفلية في phpMyAdmin تفيد بفشل مستخدم `pma` الإداري الخفي في الدخول بسبب تفعيل الباسوورد. | التجاهل التام (لا تؤثر على النظام)، أو إيقاف السطرين الخاصين بـ `controluser` و `controlpass` بملف `config.inc.php`. |

**مقرري+** هي منصة تعليمية ومهنية ثورية، لا تكتفي بتقديم المقررات الأكاديمية، بل تبني **"جسر المعرفة" (Knowledge Graph)** بين المناهج الجامعية وسوق العمل الحقيقي. تقوم المنصة باستخراج المهارات، والأدوات، والشهادات الاحترافية ديناميكياً لتوليد سير ذاتية للطلاب، وتبرز التأثير الحقيقي للخبراء المعتمدين من كبرى الشركات التقنية.

---

## 📖 الديباجة والرؤية (Vision & Mission)
وُلدت فكرة **"مقرري+"** لحل معضلة وطنية وعالمية: **الفجوة بين المناهج الأكاديمية ومتطلبات سوق العمل المتسارعة**. النظام ليس مجرد منصة تعليمية لعرض المقررات، بل هو **"نظام بيئي متكامل" (Ecosystem)** وشبكة معارف تسمح لخبراء الصناعة المعتمدين بالدخول وحقن المقررات الجامعية بمهارات وأدوات سوق العمل الحقيقية، لتوليد كفاءات هندسية جاهزة للتوظيف الفوري.

---

## 🚀 المميزات القاتلة (The Killer Features)
1. **السيرة الذاتية المدفوعة بالمهارات (Skills-Driven CV - B2C):** لا يكتب الطالب أنه "درس مقرر هندسة البرمجيات" بل يقوم النظام آلياً بتوليد سيرة ذاتية تقنية بناءً على المهارات والأدوات المستخرجة من الإثراءات.
2. **مؤشر الفجوة الأكاديمية (Academic Gap Index - B2B):** لوحة قيادة تحليلية مبنية بـ `Chart.js`، تعرض أكثر المهارات المطلوبة حالياً لتحديث المناهج.
3. **مركز قيادة الخبير (Expert Command Center):** محرر تفاعلي (Markdown) يظهر التنسيق بالوقت الفعلي مع حساب "مؤشر الأثر الطلابي" للخبير.
4. **نظام التلعيب (The Gamification Loop):** دورة تفاعل تتيح للطلاب رفع تقييم الإثراءات الممتازة (Upvote) وحفظها (Bookmark).
5. **الهيكل الأكاديمي وطلبات المقررات:** نظام هرمي معقد (جامعة ← كلية ← تخصص ← مقرر) يتيح للطلاب طلب إضافة مقررات مفقودة.
6. **واجهة القيادة العليا الموحدة (God-Eye Dashboard):** مركز مراقبة شامل لسجلات التدقيق وتوزيع الصلاحيات (Role Matrix).

## 🏛️ الهيكلة المعمارية (The Knowledge Graph Architecture)
تم بناء قاعدة البيانات كشبكة علاقات معقدة (Many-to-Many):
* **المستعمرة الأكاديمية:** `University` -> `College` -> `Major` -> `Course`.
* **المستعمرة الصناعية:** `Industry` -> `Skill` & `Tool` & `ProfessionalCertificate`.
* **نقطة التصادم (The Nexus):** كيان `CourseEnrichment` الذي يربط بين (المقرر + الخبير + المهارات + الأدوات + الشهادات).

## 🔐 الهيكلة الأمنية ومراتب الوصول (Access Matrix)
تم تقسيم النظام إلى 4 مستويات من الصلاحيات المعزولة كلياً:
* **SUPER_ADMIN:** سلطة مطلقة، سجل التدقيق (Audit Logs)، إيقاف وتفعيل الحسابات، ترقية المستخدمين، وتوليد حسابات مدراء.
* **ADMIN:** توثيق حسابات الخبراء، الموافقة/الرفض على الإثراءات وطلبات إضافة المقررات.
* **EXPERT:** إضافة إثراءات جديدة، إدارة ملفه العام (Authority Page).
* **STUDENT:** تصفح، حفظ، تقييم، استعراض السيرة التقنية، وطلب مقررات مفقودة.

## 🧪 شبكة الاختبارات المعقدة (The Test Suite)
المنصة محصنة بشبكة اختبارات تتنفذ آلياً مع عملية البناء:
* `DatabaseSeederVerificationTest`: يضمن عدم تداخل التخصصات.
* `SystemIntegrityOmniTest`: يمسح قاعدة البيانات لضمان عدم وجود بيانات أشباح (Ghost Data).
* `OmniShieldControllerTest`: يؤكد أن صفحات الإدارة العليا محرمة على بقية المستخدمين.
* `IdentityEngineSecurityTest`: يضمن خصوصية البيانات (مستخدم أ لا يمكنه تعديل بيانات ب).

## 📜 سجل التطور المعماري المعقد (The Master Changelog)

### المرحلة الأولى: تأسيس البنية التحتية والمحتوى (v1.0 - v1.19)
* **v1.0 - v1.16:** بناء الأساس، البوابة الأمنية، واعتماد المحتوى، وإضافة حلقة التفاعل ومحرك البحث الشامل.
* **v1.17 (الهيكل الأكاديمي):** تحويل الهيكلة الأكاديمية إلى هرمية (`University` → `College` → `Major` → `Course`) مع حماية الحذف وتوطين شامل للوحة الإدارة.
* **v1.18 (واجهة الطالب الهرمية):** استخدام المعالج الهرمي (Wizard) لاستكشاف المقررات عبر AJAX وتصحيح ثغرات CSRF للصفحات المجهولة.
* **v1.19 (نظام طلبات المقررات):** إضافة كيان `CourseAdditionRequest` للسماح للطلاب بطلب مقررات مفقودة، مع واجهات إدارة ومراجعة وحماية من حذف الطلبات المقبولة.

### المرحلة الثانية: الترقية المؤسسية الكبرى (Enterprise Upgrade v2.0 - v2.3)
* **v2.0 - المحرك الأعظم (Omni-Graph Data Engine):** ضخ بيانات واقعية ضخمة (53 خبيراً، 66 شهادة، 130 أداة) وتوليد 386 إثراء تقني. استئصال العشوائية عبر بروتوكول انعدام الأخطاء وبناء خوارزميات حقن حتمية لتصفية البيانات وتطهيرها.
* **v2.1 - الحماية المطلقة (The Omni-Shield):** تفعيل الخزنة الحديدية (`@PrePersist` و `@PreUpdate`) لرفض الإثراءات الخاطئة ذاتياً (Self-Healing)، وإغلاق ثغرات الصلاحيات بـ `@EnableMethodSecurity`.
* **v2.2 - عين الصقر والقيادة العليا (God-Eye):** بناء سجل مراقبة صامت (`@Auditable`) لكل العمليات، وإعادة تصميم لوحة السوبر آدمن لدمج نظام الترجمة، ومصفوفة الصلاحيات الحية، ونظام توليد المدراء الآمن بـ Flash Attributes.
* **v2.3 - محرك الهويات المزدوجة (Dual-Faced Identity Engine):** بناء خزنة خاصة للمستخدمين، وسيرة ذاتية تقنية علنية للطلاب، وواجهة "مؤشر الأثر" (Impact Score) للخبراء لتوثيق مساهماتهم.

## 🚀 دليل التشغيل السريع (Deployment Protocol)
1. **تجهيز قاعدة البيانات:** قم بإنشاء قاعدة بيانات فارغة في MySQL باسم `moqarariplus`.
2. **البناء والاختبار:** نفذ أمر `mvn clean install` لضمان عمل شبكة الاختبارات المعقدة.
3. **التشغيل:** نفذ أمر `mvn spring-boot:run`. سيقوم النظام آلياً ببناء الجداول وحقن البيانات الحقيقية بالكامل لحظة الإقلاع.
   *(المنصة مجهزة بوضع Chameleon للرفع المباشر على السحابة).*

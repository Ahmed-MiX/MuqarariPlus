🎓 مقرري+ (Muqarari+) - التوثيق التقني الشامل
"سد الفجوة بين النظرية الأكاديمية والتطبيق العملي"
مشروع برمجي متقدم تم بناؤه للمشاركة في الهاكاثون التقني. تهدف المنصة إلى ربط طلاب هندسة الحاسب والبرمجيات بخبراء الصناعة لإثراء المقررات الأكاديمية بتطبيقات عملية من سوق العمل الحقيقية.

🛠️ التقنيات المستخدمة (Tech Stack)
اللغة الأساسية: Java 17

إطار العمل (Backend): Spring Boot 3.2.4

نظام الحماية: Spring Security 6 & BCrypt Password Encoder

قواعد البيانات: MySQL 8.x & Spring Data JPA (Hibernate)

الواجهة الأمامية (Frontend): Thymeleaf, HTML5, CSS3, Bootstrap 5.3

إدارة الحزم: Maven

نظام التتبع: Git & GitHub

⚙️ متطلبات التشغيل (Prerequisites)
يجب على أي مطور في الفريق تثبيت الأدوات التالية قبل البدء:

JDK 17: (مثل Eclipse Adoptium أو Oracle JDK) وتأكد من ضبط مسار JAVA_HOME في الويندوز.

IntelliJ IDEA: النسخة المجانية (Community) أو Ultimate.

XAMPP: لتشغيل خادم قاعدة بيانات MySQL محلياً.

Git: لسحب المشروع وإدارة النسخ.

🚀 دليل التثبيت والتشغيل خطوة بخطوة
تجهيز قاعدة البيانات:

قم بتشغيل لوحة تحكم XAMPP واضغط Start بجوار MySQL.

افتح المتصفح على http://localhost/phpmyadmin.

قم بإنشاء قاعدة بيانات فارغة تماماً باسم: muqarari_plus (الترميز: utf8mb4_general_ci).

سحب المشروع:

افتح الـ Terminal واكتب: git clone [رابط-مستودع-جيت-هاب-هنا]

فتح المشروع وتحميل المكتبات:

افتح المجلد بواسطة IntelliJ IDEA.

انتظر حتى يقوم Maven بتحميل جميع المكتبات (Dependencies) من الإنترنت. إذا واجهت خطأ، تأكد من ملف pom.xml واضغط على أيقونة Reload All Maven Projects.

التشغيل:

اذهب إلى المسار: src > main > java > com > muqarariplus > platform > MoqarariPlusApplication.java.

اضغط على سهم التشغيل الأخضر (Run).

سيقوم النظام (عبر DatabaseSeeder) ببناء الجداول آلياً وزرع المقررات والمستخدمين الافتراضيين باللغتين العربية والإنجليزية.

🔐 الهيكلة الأمنية المعمارية (4-Tier Architecture)
تم بناء نظام حماية صارم يفصل بين 4 أنواع من المستخدمين، ولكل منهم طريقة دخول وصلاحيات محددة:

المالك (SUPER_ADMIN):

طريقة الدخول: باستخدام اسم المستخدم admin فقط (يُمنع الإيميل).

كلمة المرور الافتراضية: 123456_Xy

الصلاحيات: رؤية إحصائيات النظام الحية، والقدرة الحصرية على (إنشاء / حذف) حسابات الأدمنز.

المشرف (ADMIN):

طريقة الدخول: باستخدام اسم مستخدم عشوائي بصيغة adminXXXX (يُمنع الإيميل).

بيانات الدخول: يتم إنشاؤها عبر السوبر أدمن، أو عبر الـ DatabaseSeeder أثناء التشغيل الأول.

الصلاحيات: إدارة المنصة، وتغيير حالة الخبراء من PENDING إلى APPROVED.

الخبير (EXPERT):

طريقة الدخول: بالإيميل فقط.

التسجيل: يسجل حسابه بنفسه، ولكن يتم تقييده بحالة PENDING حتى يوافق عليه أحد الأدمنز.

📜 سجل التعديلات والمراحل المتقدمة (Changelog & Journey)
3. هندسة اللغات والتدويل الشامل (i18n Implementation) - [مارس 2026]
   هذه هي المرحلة الأكثر تعقيداً في عمر المشروع، حيث تم الانتقال من نظام واجهة وحيدة اللغة إلى نظام عالمي مزدوج:

تطهير الواجهات (UI Scrubbing): تم مسح كافة اللواحق اليدوية المشوهة مثل _ar و _en من 9 قوالب HTML، وتحويلها إلى مفاتيح ترجمة ديناميكية (Thymeleaf Expressions) مثل #{login.title}.

توحيد ملفات القواميس: تم بناء ملفات messages.properties و messages_ar.properties بـ 160 مفتاح ترجمة احترافي، لضمان استقرار النصوص باللغتين.

دمج المعمارية (WebConfig Refactor): تم دمج إعدادات LocaleChangeInterceptor و CookieLocaleResolver داخل ملف WebConfig.java الأصلي لحل مشكلة تعارض الـ Beans البرمجية.

دعم RTL/LTR: تم تطوير ملف layout.html ليدعم تغيير اتجاه الصفحة (RTL للعربية و LTR للإنجليزية) بشكل آلي بناءً على لغة المستخدم المحددة عبر الرابط ?lang=.

4. تحسين تجربة المستخدم (UX/UI Enhancement)
   التوافق مع Bootstrap 5: تم استبدال الخصائص الاتجاهية (مثل ml-, pr-) بالخصائص المنطقية (مثل ms-, pe-) لضمان عدم انهيار التصميم عند قلب اللغة.

البيانات الصديقة: تم تطوير الـ DatabaseSeeder ليشمل 73 مقرراً حقيقياً بخمس مستويات دراسية، مع ترجمة دقيقة لكل مقرر ووصفه لضمان ظهور الموقع بشكل مكتمل أثناء العرض.

5. مرحلة الاستقرار البرمجي الحالية
   الحالة: تم الوصول إلى نسخة "نقية" (Pure Architecture) حيث لا توجد نصوص ثابتة داخل الكود، ويتم التحكم في اللغات عبر ملفات الخصائص، مع التحضير الكامل للتحول إلى نظام المحتوى المدار كلياً من قاعدة البيانات (Dynamic Content Tables).
